대용량 분산 엑셀 데이터의 논리적 통합 및 전처리 자동화 엔진

1. 프로젝트 개요 및 배경

여러 파일과 시트에 흩어진 비정형 엑셀 데이터를 하나의 마스터 시트로 자동 통합하여 데이터 분석 전처리 시간을 획기적으로 단축한 프로젝트입니다.

  • 📅 개발 기간 : 2024.03 - 2024.04 (추정)
  • 👤 주요 역할 : Pandas 기반 데이터 병합 로직 설계, 인코딩 자동 감지 엔진 구현, 사용자 친화적 GUI 개발
  • 🚩 도입 배경 : 수작업 병합 시 발생하는 휴먼 에러, 서로 다른 인코딩(UTF-8, CP949) 혼선, 대용량 파일 처리의 한계
  • 🎯 해결 목표 : 수천만 행의 데이터를 결합해도 멈추지 않는 안정적인 멀티스레딩 통합 툴 구축

2. 기술 스택 및 환경

분류 적용 기술 및 도구 도입 목적
언어 Python 3.10 강력한 데이터 처리 라이브러리 생태계 활용
데이터 분석 Pandas, NumPy 고속 대용량 행렬 연산 및 데이터프레임 조작
멀티태스킹 Threading UI 프리징 방지 및 백그라운드 연산 처리
엔진 openpyxl, xlrd 신/구형 엑셀 포맷(.xlsx, .xls) 범용성 확보

4. 주요 해결 전략

전략 1: [이중 인코딩 폴백 시스템]

CSV 파일 로드시 UTF-8 실패 시 CP949로 즉시 재시도하는 예외 처리 방어 루틴을 통해 한글 깨짐 문제를 완전 해결했습니다.

전략 2: [멀티스레딩 기반 UI 안정화]

Pandas의 연산 작업과 GUI 루프를 물리적으로 분리하여, 수십만 건의 데이터 처리 중에도 프로그램이 '응답 없음' 상태에 빠지지 않도록 설계했습니다.

6. 트러블슈팅 및 성장

Pandas 데이터프레임의 인덱스 관리 이슈를 `ignore_index=True` 옵션 최적화로 해결하며 데이터 정합성의 중요성을 깨달았습니다. 대량 데이터 연산 시 메모리 효율을 고려한 설계 방식을 학습했습니다.
